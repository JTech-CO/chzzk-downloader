# Chzzk Downloader v2.2.5 업데이트 안내

안녕하세요, 치지직 다운로더 개발자입니다.

이번 **v2.2.5**는 사용자 제보로 확인된 라이브 다시보기 HLS 다운로드 차단과 `inKey` 없는 영상의 Windows·팟플레이어 재생 호환성 문제를 수정한 업데이트입니다.

---

## 1. "HLS URL은 Naver/Pstatic 도메인만 허용됩니다" 오류 수정

### 원인

v2.2.4에서 백그라운드 URL 검증을 Naver/Pstatic 도메인으로 제한했지만, 치지직 공식 영상 상세 API는 일부 라이브 다시보기 HLS를 다음 CDN에서 전달합니다.

- `light-slit.akamaized.net`
- `ex-nlive-slitvod-streaming.navercdn.com`

정상적인 치지직 HLS URL도 다운로드 시작 전에 차단되는 과도한 제한이었습니다.

### 수정

- 매니페스트에 위 두 CDN 호스트만 추가
- 코드에서는 해당 호스트의 `/chzzk/` 경로만 허용
- 마스터·화질별 플레이리스트·초기화 조각·미디어 조각 및 리다이렉트 후 응답 URL을 단계마다 다시 검증
- HTTPS, 공개 주소, 치지직 탭 발신자 제한은 그대로 유지
- Referer/Origin 변경 규칙은 기존처럼 Naver/Pstatic 요청에만 적용

`<all_urls>`나 `*.akamaized.net` 전체 권한으로 되돌리지 않고 실제 치지직 재생 경로만 열었습니다.

---

## 2. `inKey` 없는 VOD의 팟플레이어 무한 로딩·길이 미표시 수정

### 원인

`inKey` 없는 라이브 다시보기는 HLS/fMP4 조각으로 제공됩니다. 실제 제보 영상의 초기화 조각을 확인한 결과 `moov`는 존재했지만 다음 값이 모두 0이었습니다.

- 전체 영상 재생시간(`mvhd`)
- 영상·오디오 트랙 재생시간(`tkhd`, `mdhd`)
- 조각 재생 시작 시각이 0초가 아닌 원래 방송 타임라인 기준
- 파일 끝의 조각 탐색 인덱스(`mfra`) 없음

이 때문에 Windows Media Player는 순차 재생만 가능하고, Windows 탐색기는 길이를 표시하지 못하며, 팟플레이어는 파일 분석 또는 탐색 과정에서 무한 로딩될 수 있었습니다.

### 수정

- HLS의 `#EXTINF`를 합산해 실제 재생시간 계산
- `mvhd`, `tkhd`, `mdhd`, `mehd` duration 기록
- 영상과 오디오 사이의 원래 시간 차이는 유지하면서 모든 `tfdt`를 0초 기준으로 보정
- 4GB를 넘는 장시간 파일도 처리할 수 있도록 64비트 `tfra`/`mfra` 탐색 인덱스 생성
- 메모리 병합과 OPFS 디스크 스트리밍 양쪽에 같은 최종화 적용

영상과 음성을 재인코딩하지 않고 MP4 메타데이터만 완성하므로 화질 손실이 없고, 10~12시간급 VOD도 기존 디스크 스트리밍 경로로 계속 다운로드합니다. 영상 길이를 이유로 다운로드를 중단하지 않습니다.

---

## 3. 검증 결과

실제 `inKey` 없는 치지직 VOD의 초기화 조각과 HLS 세그먼트로 확인했습니다.

- 전체·영상·오디오 duration이 HLS 재생시간과 일치
- 시작 타임라인이 오디오는 0초, 영상은 원래 A/V 차이를 유지한 0.041초로 보정
- 파일 끝에 `mfra` 인덱스 생성
- Windows 탐색기 속성 처리기에서 길이 인식
- `ffprobe`에서 MP4/H.264/AAC 스트림 인식
- 중간 시점 탐색 후 디코딩 성공

---

## 4. 장시간 VOD 안전성 유지

v2.2.4의 다음 보호 장치는 그대로 유지됩니다.

- Range 요청은 `206 Partial Content` 응답만 처리
- 예상 Range 크기와 응답 본문 크기 검증
- 비정상적으로 큰 단일 세그먼트 차단
- 큰 영상은 OPFS에 순차 기록해 메모리 사용량 제한
- 로컬호스트·사설망 URL 및 비정상 메시지 차단

## 업데이트 방법

기존 설치본을 **v2.2.5**로 업데이트해 주세요. 개발자 모드로 설치한 경우 최신 `chzzk-downloader.zip`의 압축을 해제한 뒤 확장 프로그램 관리 화면에서 다시 로드하면 됩니다.
